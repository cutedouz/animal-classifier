# Phase 4H-4c：教師作答檢視儀表板

## 背景與目的

Phase 4H-4a 與 4H-4b 讓教師可以整理班級名冊、修正學生資料與管理學生登入資訊。名冊修正後，教師需要一個安全的唯讀入口，確認學生個人任務與小組任務是否已開始、是否仍為草稿、是否已提交終稿，以及是否曾取得 AI 回饋。

本階段新增的「學生作答檢視」只解決檢視需求，不改變學生提交流程，也不加入評分或教師回饋功能。

## 教師可以看到的內容

- 個人任務作答狀態：尚未開始、草稿、已提交。
- 小組任務作答狀態：尚未開始、草稿、已提交。
- 終稿內容：若已有終稿，優先顯示終稿作答內容。
- 草稿標籤：若目前只有草稿，頁面會明確標示「尚未提交終稿」；若終稿後又有新草稿，也會提示教師目前優先檢視終稿。
- 終稿提交時間：只使用最新終稿版本的提交時間。
- AI 回饋狀態：只顯示是否曾取得 AI 回饋、最近回饋時間與回饋狀態。

## 教師不能做的事

- 不能編輯學生或小組作答。
- 不能評分。
- 不能退回作答。
- 不能留下教師評語。
- 不能檢視研究識別碼，例如 participantCode 或 anonymousCode。
- 不能看到密碼雜湊、帳號秘密、原始 JSON 或 AI 原始回應內容。

## 權限與隱私規則

- 教師必須具有啟用中的 TeacherProfile。
- 班級必須屬於該 TeacherProfile，否則 loader 會拒絕載入。
- 儀表板只納入班級中可見的 course task settings。
- UI 可在路由參數中使用必要的內部 id，但畫面上的標籤必須使用班級名稱、座號、學生姓名、小組名稱與任務名稱等人類可讀資訊。
- 作答內容會依 TaskField 的 fieldLabel 與 fieldName 轉為安全欄位列，不直接輸出 raw JSON。
- 敏感鍵值會被過濾，包括 password、token、participantCode、anonymousCode、userId、studentProfileId、groupId、rosterStudentId、email、hash 與 rawResponseJson。

## 與後續階段的關係

- 教師回饋與評分：留待後續階段設計，不在本階段提供。
- 匯出功能：留待後續階段處理。
- G0-G4 研究條件 UI：留待後續階段處理。
- SRL 頁面：留待後續階段處理。
- 前後測流程：不屬於本階段範圍。

## 非目標

- 不新增資料庫 schema。
- 不新增 migration。
- 不變更學生提交行為。
- 不提供教師修改、退回、評論、評分或匯出作答的能力。
- 不顯示研究識別碼、帳號秘密、原始 JSON 或 AI 原始回應。
